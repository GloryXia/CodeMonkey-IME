# 规则引擎设计文档

## 概述

Hybrid IME 的规则引擎是一个由多层规则组成的决策系统。每一层规则独立运作，可单独开关。

## 规则层级

```
Layer 0: 保护规则 （最高优先级，一票否决）
Layer 1: 标点决策规则
Layer 2: 自动空格规则
Layer 3: 候选重排规则
```

## Layer 0 — 保护规则

**职责**：识别不应被任何智能变换影响的内容。

### 保护对象

| 模式 | 检测方式 | 示例 |
|:-----|:---------|:-----|
| URL | 前缀匹配 `https?://` | `https://example.com/api/v1` |
| 文件路径 | 前缀匹配 `~/` `./` `../` `/` | `~/Downloads/test.txt` |
| 邮箱 | 正则 `\w+@\w+\.\w+` | `user@example.com` |
| CLI 参数 | 前缀匹配 `--` `-` | `--force` `-v` |
| 变量名 | snake_case / camelCase | `user_id` / `refreshToken` |
| 版本号 | 正则 `v?\d+\.\d+` | `v1.2.3` |

### 保护行为

- 禁止标点自动替换
- 禁止自动空格
- 禁止候选重排干预

## Layer 1 — 标点决策规则

**职责**：决定半角标点是否替换为中文全角标点。

### 决策流程

```
输入标点字符 (,.?!:;)
  │
  ├─ 保护模式? → 不替换
  ├─ 开关关闭? → 不替换
  ├─ 组合态中? → 不替换（交给 Rime 处理）
  │
  ├─ 上一字符类型:
  │   ├─ 中文 → 替换为全角
  │   ├─ 英文 → 检查语言上下文
  │   │   ├─ 中文/混合语境 → 替换
  │   │   └─ 英文语境 → 不替换
  │   ├─ 数字 → . 和 , 不替换（可能是小数/千分位）
  │   ├─ 标点 → 不替换
  │   └─ 未知 → 不替换（保守）
  │
  └─ 记录操作（用于撤销）
```

### 映射表

| 输入 | 中文全角 |
|:-----|:---------|
| `,` | `，` |
| `.` | `。` |
| `?` | `？` |
| `!` | `！` |
| `:` | `：` |
| `;` | `；` |

### MVP 不自动替换

| 符号 | 原因 |
|:-----|:-----|
| `'` | 单引号/撇号歧义大 |
| `"` | 引号配对复杂 |
| `()` | 代码/数学表达式 |
| `...` | 省略号需特殊处理 |
| `--` | 命令参数/破折号 |

## Layer 2 — 自动空格规则

**职责**：在中英文混排时自动补充空格。

### 规则

| 前文 | 候选 | 动作 | 示例 |
|:-----|:-----|:-----|:-----|
| 英文/数字 | 中文 | 前加空格 | `bug` + `很难` → `bug 很难` |
| 中文 | 英文/数字 | 前加空格 | `修复` + `bug` → `修复 bug` |
| 中文 | 中文 | 不加 | `你好` + `世界` |
| 英文 | 英文 | 不加 | `hello` + `world` |

### 保护场景（不加空格）

- 路径中
- URL 中
- 变量名中
- 命令参数中
- 版本号中

## Layer 3 — 候选重排规则

**职责**：根据上下文调整候选排序。

### 评分加权

| 条件 | 加分 |
|:-----|:-----|
| 中文语境 + 中文候选 | +10 |
| 英文语境 + 英文候选 | +10 |
| 技术术语匹配 | +5~8 |
| 输入长度完全匹配 | +3 |
| 中文语境 + 非术语英文 | -5 |
| 英文语境 + 中文候选 | -5 |
